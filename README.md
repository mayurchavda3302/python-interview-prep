## 🧠 Daily Rules (Follow Every Day)

- ✅ Solve **3–5 questions daily** minimum
- ✅ Write code from scratch — no copy-paste
- ✅ Always test edge cases (empty input, None, 0, negatives)
- ✅ After solving, read your code once more and simplify
- ✅ Maintain a "mistakes journal" — note where you got stuck
- ✅ Spend 20% time on theory, 80% on coding

## 💡 Tips for the Actual Interview

- **Think out loud** — Explain what you're doing as you code
- **Ask about edge cases** — Shows maturity
- **Start with brute force**, then optimize
- **Don't panic on unknown questions** — Break them into smaller pieces
- **Know your time/space complexity** — Even basic Big-O awareness helps
